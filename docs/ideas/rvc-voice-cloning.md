# Идея: клонированный голос для TTS через RVC

## Что это

RVC (Retrieval-based Voice Conversion) — конвертер голоса, а не text-to-speech.
На вход подаётся уже готовый звук, модель меняет тембр/окраску на целевой голос,
сохраняя интонацию и темп исходника. Обучается на 10-30 минутах чистой записи
целевого голоса.

Для наших задач RVC не используется отдельно, а ставится вторым шагом после
существующего `edge-tts`:

```
текст сообщения/алерта → edge-tts (как сейчас) → WAV → RVC-конвертация → проигрывание
```

## Зачем

Сейчас TTS в `tts_worker.py` читает чат/алерты стандартными голосами edge-tts
(`ru-RU-DmitryNeural` и т.п.). Идея — прогонять этот же звук через RVC-модель,
обученную на конкретном голосе (свой голос стримера или готовая модель), чтобы
чат/донаты озвучивались этим голосом вместо дефолтного.

## Как это ляжет на архитектуру

1. `tts_worker.py`: между генерацией WAV через edge-tts и проигрыванием (MCI)
   добавляется шаг постобработки через RVC.
2. RVC-модель (`.pth` + `.index`) грузится один раз при старте процесса
   (отдельный локальный сервис/подпроцесс), а не на каждое сообщение —
   иначе долгий cold start на каждое сообщение.
3. В `/settings` (`overlay_server.py`, `overlay/settings.html`) добавляется
   тумблер "клонированный голос вкл/выкл" — по аналогии с остальными
   настройками, применяется на лету через `runtime_settings.py`.
4. Можно ограничить RVC только алертами (донаты/сабы/рейды), оставив обычный
   чат на чистом edge-tts — событий меньше, риска для производительности
   меньше.

## Ресурсы (3060 Ti, стрим CS)

RVC-конвертация одного сообщения — короткий импульс (~1-2GB VRAM на секунду-
две), а не постоянно висящий процесс. Это принципиально другой профиль
нагрузки, чем локальная LLM, крутящаяся весь стрим параллельно с игрой.
Если нужно вообще исключить риск для FPS — можно гонять RVC на CPU
(медленнее, конвертация занимает пару секунд, зато ноль нагрузки на видеокарту).

## Варианты старта (на выбор)

- **Демо-модель**: ставим RVC, проверяем пайплайн edge-tts → RVC → плейбек на
  готовой чужой модели голоса. Быстро, но голос не свой.
- **Готовая модель**: если уже скачан `.pth`/`.index` откуда-то — просто
  подключаем, обучение не нужно.
- **Свой голос**: нужно 10-30 минут чистой записи (без музыки/шума),
  тренировка модели — разовая, оффлайн, не во время стрима.

## Статус

Идея зафиксирована, реализация не начата. Ждём решения — чей голос
использовать (см. варианты выше) — и приступаем к интеграции в
`tts_worker.py` + `overlay/settings.html`.
