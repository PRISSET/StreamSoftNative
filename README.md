# StreamSoft

Софт для стримера: слушает чат **YouTube** и **Twitch** в реальном времени, озвучивает
сообщения лёгким TTS (`edge-tts`, без GPU и тяжёлых моделей), пересылает всё в **Telegram**
с пометкой платформы, показывает красивый **оверлей для OBS** с историей чата и алертами
подписок/фоллоу/рейдов/донатов (через Twitch EventSub), плюс веб-панель для настройки всего
этого на лету.

## Возможности

- **Чат YouTube + Twitch** в одном потоке, с указанием платформы.
- **TTS-озвучка** сообщений — голос автоматически выбирается по языку сообщения (RU/EN),
  без сторонних тяжёлых движков (используется `edge-tts` + встроенный Windows MCI для
  воспроизведения).
- **Пересылка в Telegram** с пометкой 🔴 YouTube / 💜 Twitch.
- **Управление прямо из Telegram**: `/mute`, `/unmute`, `/muted`, `/skip`, `/volume`.
- **Алерты Twitch EventSub**: фоллоу, подписка, подарочная подписка, рейд, донат битсами —
  читаются голосом, летят в Telegram и показываются на оверлее.
- **Оверлей для OBS** (Browser Source):
  - `/chat` — только чат, `/events` — только алерты, `/` — всё вместе.
  - История последних сообщений чата сохраняется и подхватывается при обновлении источника.
  - 3 готовых шаблона оформления: Minimal, Card, Neon.
  - Свои гифки и звуки под каждое событие (см. `overlay/media/`).
- **Веб-панель настроек** (`/settings`): живое превью оверлея с тестовыми алертами,
  регулировка голосов/скорости/громкости TTS, громкости звука алертов, список замьюченных,
  загрузка gif/mp3 прямо из браузера — всё применяется сразу, без перезапуска.

## Установка

```powershell
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

Скопируйте `.env.example` в `.env` и заполните нужные поля:

```powershell
Copy-Item .env.example .env
```

Можно включить только одну платформу — оставьте поля второй пустыми.

## YouTube: где взять API key и video ID

1. [Google Cloud Console](https://console.cloud.google.com/) → создайте проект → включите
   **YouTube Data API v3** → создайте **API key** → в `YOUTUBE_API_KEY`.
2. `YOUTUBE_VIDEO_ID` — ID видео трансляции или просто вся ссылка (ID вытащится сам).
   Трансляция должна быть уже запущена (активный live chat).

## Twitch: регистрация приложения

1. [dev.twitch.tv/console/apps](https://dev.twitch.tv/console/apps) → **Register Your
   Application**. OAuth Redirect URL — `https://localhost` (не используется, но поле
   обязательно). Тип клиента — **Публичный** (Public).
2. Client ID → в `TWITCH_CLIENT_ID`. Client Secret не нужен.
3. `TWITCH_CHANNEL` — ник канала без `#`.

При первом запуске программа выведет ссылку и код авторизации в консоль — подтвердите один
раз в браузере (нужны права на чтение чата и на алерты EventSub). Токен сохранится в
`twitch_token.json` и обновляется автоматически.

## Telegram

1. Токен бота — от `@BotFather`.
2. `chat_id`: напишите боту любое сообщение, затем откройте
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates` и найдите `"chat":{"id": ...}`.

## Запуск

```powershell
python main.py
```

- Чат/события выводятся в консоль.
- Оверлей: `http://127.0.0.1:8099/` (или `/chat`, `/events`).
- Панель настроек: `http://127.0.0.1:8099/settings`.

## Настройка через `.env`

См. `.env.example` — там прокомментированы все переменные: ключи YouTube/Twitch/Telegram,
включение EventSub и управления через Telegram, голоса и параметры TTS, порт оверлея.
Часть этих же параметров (голоса, громкость, тема оверлея) можно менять на лету через
`/settings`, они сохраняются в `runtime_settings.json` и переживают перезапуск.

## Свои гифки и звуки для алертов

Кладите файлы в `overlay/media/` (или загружайте через `/settings` → «Алерты и медиа»)
строго с именами `follow.gif`/`follow.mp3`, `subscribe.gif`/`subscribe.mp3`,
`gift_sub.gif`/`gift_sub.mp3`, `raid.gif`/`raid.mp3`, `cheer.gif`/`cheer.mp3`. Если файла
нет — просто не покажется/не прозвучит, без ошибок.

## Заметки

- Воспроизведение TTS и звуков алертов рассчитано на Windows (использует `ctypes`/MCI и
  браузерный `<audio>` в оверлее).
- Чтобы голос TTS попал в OBS отдельным треком — направьте вывод процесса python в
  виртуальный аудиокабель (например VB-CABLE) и добавьте его как Audio Input Capture в OBS.
  Звук алертов на оверлее OBS ловит сам, без доп. настройки (это звук браузер-источника).
