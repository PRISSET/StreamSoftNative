# Changelog

## 1.0.0

- Чат-агрегация (Twitch/YouTube), веб-оверлей, авто-подключение к OBS,
  Discord Rich Presence, трей, Check&Install для TTS/RVC-модулей.
- RVC ставится через живую pip-установку прямо на машине пользователя, под
  его CUDA (торч подбирается под реальную версию CUDA-драйвера через NVML).
- Core сам запускает RVC-адаптер — не нужно вручную запускать `start.bat`.
- TTS и RVC оживают сразу после установки модуля — без перезапуска
  приложения.
- Автообновление: приложение само проверяет новые версии на GitHub и тихо
  ставит их в фоне, без ручного скачивания инсталлятора.
- Вкладка «Обновления» — показывает установленную версию и список релизов
  с описанием каждого.
- Даккинг громкости: при озвучке чата/алертов остальные приложения (игра,
  музыка) автоматически приглушаются и возвращаются на место после.
  Настраивается на вкладке «Голос».
- Опрос в чате: зрители голосуют командами !1/!2/... прямо в чате, результат
  сразу на оверлее (`/poll` источник в OBS). При старте опроса бот сам
  объясняет в чате, как голосовать.
- Музыка по баллам: свои баллы (не Twitch Channel Points) начисляются за
  активность в чате, тратятся командой `!song <ссылка YouTube/SoundCloud>`.
  Плеер — отдельный источник `/nowplaying` в OBS. Стример (владелец канала)
  заказывает музыку бесплатно.
- `!help` — список команд прямо в чате, `!points` — проверить баланс баллов.
  Раз в 15 минут бот ненавязчиво напоминает про баллы и `!song`, если фича
  включена.
- Источники Chat/Alerts/Poll/Now Playing в OBS создаются нативным небольшим
  размером вместо растяжения на весь канвас — можно свободно двигать/
  масштабировать без потери качества.
- Переключатели и слайды в iOS-стиле, акцентный зелёный цвет по всему UI.
- Программа пишет лог (`streamsoft.log` рядом с настройками) — проблемы с
  чатом/RVC/музыкой теперь диагностируются без специальной отладочной сборки.
